Guidelines for Developers and QA Teams

This page complements the paper "Verification of ROS-based Systems: Guidelines for Developers and QA Teams" and is an online repository of the guideline catalog for ROS projects.

You can further find the guidelines and information on the evaluation process. Reproduction kits, specifications and accompanying code can be found in repositories.

Guideline Catalog

Guidelines



Constraint identification

  1. CI1

    Identify timing constraints

  2. CI2

    Identify security and privacy constraints

  3. CI3

    Identify safety constraints

Code design and implementation for FT & RV

  1. CD1

    Strive for ROS nodes with single responsibility

  2. CD2

    Ensure global time monotonicity of events and states

Instrumentation for FT & RV

  1. I1

    Provide an API for querying and updating internal lifecycle.

  2. I2

    Provide an API for logging and filtering

  3. I3

    Provide an API for injecting faults in execution scenarios

  4. I4

    Isolate components for testing

Prepare execution environment for FT & RV

  1. PE1

    Understand the overhead acceptance criteria

  2. PE2

    Create models for runtime assessment

Specify (un)desired behavior

  1. SDB1

    Property specification using a logic-based language

  2. SDB2

    Use Domain Specific Languages to Specify Properties

  3. SDB3

    Use languages and tools to scenario-based specification of test cases

Generate monitors and test cases

  1. MTA1

    Improve the robustness of the system by performing noise and fault injection

  2. MTA2

    Exploit automation for Test Case Generation, Test Case Prioritization, Test Oracle Specification

System execution for FT & RV

  1. SE1

    Use record-and-playback when performing ex- ploratory field tests

  2. SE2

    No GUIs! Prioritize headless simulation

Analysis and Reporting

  1. AR1

    Perform postmortem analysis to diagnose non- passing test cases

  2. AR2

    Use reliable tooling in order to manage field data
